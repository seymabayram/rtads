# Contributing to RTADS 🛡️

Thank you for your interest in RTADS! We follow a standard contribution process to ensure the quality and stability of our real-time anomaly detection system.

## 🚀 How to Contribute

1.  **Fork the repository** (highly recommended).
2.  **Create a branch** for your feature or bugfix (`feat/new-feature` or `fix/issue-description`).
3.  **Make your changes** following the project's coding standards.
4.  **Write or update tests** in the `tests/` directory if necessary.
5.  **Submit a Pull Request** with a clear description of your changes.

## 🧪 Development Workflow

-   **Environment Setup**: Use the provided `scripts/setup_dev.sh` to initialize your local environment.
-   **Coding Standards**: Use PEP8 for Python code and ensure meaningful commit messages.
-   **Testing**: Run tests using `python3 -m unittest discover tests`.

## 🤝 Code of Conduct

We are committed to providing a welcoming and inspiring community for all. Please be respectful and professional in your interactions.
